## 2) Multiple Inputs
File:
- day-008/Positional vs Keyword Arguments/task.py

Key learning:
- A function can take multiple inputs separated by commas.
- Positional arguments depend on order.
- Keyword arguments remove confusion by explicitly mapping values to parameter names.

Example used:
- greet_with("Adarsh Sharma", "India")  (positional)
- greet_with(location="India", name="Adarsh Sharma") (keyword)

## 3) Caesar Cipher 1 (Encryption)
File:
- day-008/Caesar Cipher 1/task.py

What was done:
- Built encrypt(original_text, shift_amount).
- Shifted each letter forward using alphabet index.
- Wrap-around fix using modulo:
  - (index + shift) % len(alphabet)
- Added a check so non-alphabet characters (spaces/symbols/numbers) are kept as-is.

Key learning:
- Use modulo to prevent index out of range.
- Using len(alphabet) makes it flexible and clearer.

---

## 4) Caesar Cipher 2 (Decryption + Direction)
File:
- day-008/Caesar Cipher 2/task.py

What was done:
- Built decrypt(original_text, shift_amount) by shifting backwards:
  - (index - shift) % len(alphabet)
- Kept encrypt() also.
- Used direction input (encode/decode) to call the right function.

Key learning:
- Decrypt is reverse shift.
- Same modulo wrap-around logic applies.

## 5) Caesar Cipher 3 (Final Version)
Files:
- day-008/Caesar Cipher 3/task.py
- day-008/Caesar Cipher 3/art.py

What was done:
- Imported and printed logo at start:
  - from art import logo
- Combined encode/decode into one function:
  - caesar(original_text, shift_amount, encode_or_decode)
- Decode implemented by reversing shift:
  - if decode: shift_amount *= -1
- Non-letter handling:
  - if letter not in alphabet: keep it unchanged
- Restart feature:
  - while keep_playing: ask inputs again, run cipher, ask user to continue

Key learning:
- One reusable function is cleaner than two separate functions.
- Restart loop makes the program interactive.
- Handling non-letters avoids crashes and supports real messages.
