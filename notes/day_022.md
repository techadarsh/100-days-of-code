## 1) main.py (Game Controller)
What main.py did:
- Created the screen:
  - size 800x600
  - black background
  - title "Pong"
  - used `screen.tracer(0)` for smooth animation (manual screen updates)

- Created objects:
  - `r_paddle = Paddle((350, 0))`
  - `l_paddle = Paddle((-350, 0))`
  - `ball = Ball()`

- Added keyboard controls:
  - Right paddle:
    - Up key -> move up
    - Down key -> move down
  - Left paddle:
    - 'w' -> move up
    - 's' -> move down

- Game loop:
  - while game_is_on:
    - time.sleep(0.1)
    - screen.update()
    - ball.move()

Key takeaway:
- tracer(0) + screen.update() gives smoother animation.

## 2) paddle.py (Paddle Class)
Goal:
- Represent each paddle as a Turtle object.

Implementation:
- Paddle class inherits from Turtle:
  - `class Paddle(Turtle):`

Setup:
- shape "square"
- color white
- stretched into a rectangle:
  - shapesize(stretch_wid=5, stretch_len=1)
- penup() so no drawing lines
- goto(position) to place paddle

Methods:
- `go_up()`
  - increases y by MOVE_DISTANCE
- `go_down()`
  - decreases y by MOVE_DISTANCE

Key takeaway:
- Inheritance lets Paddle directly use Turtle movement methods.

## 3) ball.py (Ball Class)
Goal:
- Represent the ball as a moving Turtle object.

Implementation:
- Ball class inherits from Turtle:
  - `class Ball(Turtle):`

Setup:
- shape "circle"
- color white
- penup()
- starts at center

Movement:
- Ball has x_move and y_move values (starting small, e.g. 10)
- `move()` updates position using:
  - new_x = xcor() + x_move
  - new_y = ycor() + y_move
  - goto(new_x, new_y)

Bounce logic (basic):
- Later will be extended for wall/paddle collisions

Key takeaway:
- Ball movement is controlled by changing x_move/y_move direction.
