## 1) Files: Reading
File:
- days/day-024/Files, Directories and Paths/main.py

Key learning:
- Open a file using `open()`:
  - file = open("my_file.txt")
- Read content:
  - contents = file.read()
- Always close after use:
  - file.close()

Better approach (auto close):
- Using `with`:
  - with open("my_file.txt") as file:
      contents = file.read()

Why `with` is better:
- automatically closes the file
- cleaner and safer

---

## 2) Files: Writing / Appending
Key learning:
- Write mode overwrites file:
  - with open("my_file.txt", mode="w") as file:
      file.write("New text")

- Append mode adds to end:
  - with open("my_file.txt", mode="a") as file:
      file.write("\nMore text")

- If file doesn't exist, "w" creates it.

## 3) Paths
- Relative path: based on current working directory
  - "Input/Names/invited_names.txt"
- Absolute path: full system path
  - "/Users/adarsharma/projects/..."

Key takeaway:
- Use relative paths when possible for portability.

## 4) Mail Merge Project
File:
- days/day-024/Mail Merge Project Start/main.py

Goal:
- Read a list of names.
- Read a template letter.
- Replace `[name]` placeholder with each person's name.
- Save each personalised letter into Output folder.

Steps implemented:
1. Read names from:
   - `./Input/Names/invited_names.txt`
2. Read letter template from:
   - `./Input/Letters/starting_letter.txt`
3. For each name:
   - remove newline using `.strip()`
   - replace placeholder:
     - `letter.replace("[name]", name)`
4. Create output file for each name:
   - `./Output/ReadyToSend/letter_for_{name}.txt`
5. Write personalised content into that file.

Key takeaway:
- File handling + string replace can automate repetitive writing tasks.
