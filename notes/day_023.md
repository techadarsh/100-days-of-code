# Day 023 Notes

## Project: Turtle Crossing (Capstone)

Today I built a Turtle Crossing game using Turtle + OOP. The player (a turtle) must cross the road while avoiding moving cars. Each successful crossing increases the level and makes cars faster.

Project files (Day 23):
- days/day-023/The Turtle Crossing Capstone Project/main.py
- days/day-023/The Turtle Crossing Capstone Project/player.py
- days/day-023/The Turtle Crossing Capstone Project/car_manager.py
- days/day-023/The Turtle Crossing Capstone Project/scoreboard.py

---

## 1) main.py (Game Controller)
What main.py does:
- Sets up the screen:
  - width=1000, height=800
  - screen.tracer(0) to control animation manually
- Creates objects (instances):
  - car = CarManager()
  - score = Scoreboard()
  - player = Player()
- Sets keyboard control:
  - Up arrow calls player.move

Game loop:
- Runs while game_is_on is True:
  - time.sleep(0.1) (controls speed)
  - screen.update() (renders one frame)
  - car.create_car() (randomly spawns cars)
  - car.move() (moves all cars)

Collision detection:
- Loops through all cars:
  - if each_car.distance(player) < 20:
    - game ends (game_is_on = False)
    - score.game_over()

Successful crossing:
- If player.is_at_finish_line() is True:
  - player.reset_position() (turtle goes back to start)
  - car.level_up() (cars become faster)
  - score.increase_score() (level increases)

---

## 2) player.py (Player class)
Class:
- Player inherits from Turtle:
  - class Player(Turtle)

Constants:
- STARTING_POSITION = (0, -380)
- MOVE_DISTANCE = 10
- FINISH_LINE_Y = 380

Attributes / setup:
- shape is set to "turtle"
- penup() to avoid drawing lines
- heading set to 90 degrees (facing up)

Methods:
- reset_position()
  - moves turtle to STARTING_POSITION
- move()
  - forward(MOVE_DISTANCE) when Up key pressed
- is_at_finish_line()
  - returns True if ycor() > FINISH_LINE_Y else False

---

## 3) car_manager.py (CarManager class)
Purpose:
- Manages all car objects and their movement.

Constants:
- COLORS = ["red", "orange", "yellow", "green", "blue", "purple"]
- STARTING_MOVE_DISTANCE = 5
- MOVE_INCREMENT = 10

Attributes:
- self.all_cars
  - list that stores all car Turtle objects

Methods:
- create_car()
  - uses random chance:
    - random.randint(1, 6)
    - only creates a car when chance == 1
  - creates a Turtle("square") car:
    - stretches shape (length=2, width=1)
    - picks random color
    - spawns at x=500 and random y between -300 and 300
  - appends the new car into self.all_cars

- move()
  - moves every car leftwards using:
    - car.backward(STARTING_MOVE_DISTANCE)

- level_up()
  - increases car speed by incrementing STARTING_MOVE_DISTANCE:
    - STARTING_MOVE_DISTANCE += MOVE_INCREMENT
  - (This uses global STARTING_MOVE_DISTANCE to increase difficulty each level.)

---

## 4) scoreboard.py (Scoreboard class)
Class:
- Scoreboard inherits from Turtle:
  - class Scoreboard(Turtle)

Setup:
- hidden turtle (hideturtle)
- color black
- score starts at 1 (Level 1)
- positioned at top-left: (-450, 360)

Methods:
- update_scoreboard()
  - clears previous text and writes:
    - "Level: {score}"
- increase_score()
  - score += 1 then update_scoreboard()
- game_over()
  - prints "GAME OVER" in the center of the screen

---

## Key Takeaways
- OOP helped split responsibilities:
  - Player controls movement and finish detection
  - CarManager handles car spawning, movement, and speed scaling
  - Scoreboard handles level display and game over message
- Collision detection is easy using Turtle’s distance() method.
- screen.tracer(0) + screen.update() gives smooth animation control.
- Level system makes the game progressively harder by increasing car speed.
