## 2) Using OOP: Snake class (snake.py)
### Why a class here?
Instead of keeping multiple segment turtles in one big file, I created a Snake class to manage:
- snake segments list
- snake head
- movement logic
- direction controls

### Attributes used
- self.segments
  - list of Turtle objects (each Turtle is one snake block)
- self.head
  - first segment (self.segments[0]) used for heading and forward movement

### Constants (readability + control)
- STARTING_POSITION = [(0, 0), (-20, 0), (-40, 0)]
- MOVE_DISTANCE = 20
- UP = 90, DOWN = 270, LEFT = 180, RIGHT = 0

Key takeaway:
- Using constants makes movement and directions easier to control and avoids magic numbers.

---

## 3) Creating the snake body (create_snake)
Method:
- create_snake()

Logic:
- Loop through STARTING_POSITION
- For each position:
  - create a Turtle square segment
  - set color white
  - penup() to prevent drawing lines
  - goto(position)
  - append to self.segments

Key takeaway:
- Each snake block is a separate Turtle object stored inside a list.

---

## 4) Moving the snake (move)
Method:
- move()

Logic:
- Move from the last segment to the first (reverse loop):
  - each segment copies the coordinates of the segment in front of it
- Finally move the head forward by MOVE_DISTANCE

Key takeaway:
- Reverse loop is important so you donâ€™t overwrite positions while moving.

## 5) Keyboard controls (main.py + Snake methods)
In main.py:
- screen.listen()
- screen.onkey(snake.up, "Up")
- screen.onkey(snake.down, "Down")
- screen.onkey(snake.left, "Left")
- screen.onkey(snake.right, "Right")

In snake.py:
- up/down/left/right methods change the head direction using setheading()

Safety rule added:
- Snake cannot reverse directly:
  - if heading is DOWN, cannot go UP (and vice versa)
  - if heading is RIGHT, cannot go LEFT (and vice versa)

Key takeaway:
- This prevents instant self-collision type moves and matches real snake game behavior.
